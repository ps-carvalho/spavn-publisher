# 📚 Publisher User Guide

Welcome to the **Publisher User Guide** — your comprehensive resource for managing content, media, users, and integrations in Publisher CMS.

Publisher is a developer-first headless CMS that provides an intuitive admin interface for content editors while offering powerful APIs for developers. This guide covers everything you need to know to effectively use Publisher for your content management needs.

---

## 🚀 Quick Start

Get up and running with Publisher in just a few steps:

| Step | Action | Details |
|------|--------|---------|
| 1 | **Access the Admin** | Navigate to `/admin` in your browser |
| 2 | **Log In** | Use your credentials provided by your administrator |
| 3 | **Explore Content Types** | View available content types from the dashboard |
| 4 | **Create Content** | Click on a content type and start creating entries |

> 💡 **New to Publisher?** Start with the [Getting Started](./Getting-Started/) section for a detailed walkthrough.

## 👤 User Roles & Permissions

Publisher uses a role-based access control (RBAC) system. Your role determines what actions you can perform:

| Role | Description | Capabilities |
|------|-------------|--------------|
| **Super Admin** | Full system access | All features including system configuration and user management |
| **Admin** | Content & user administrator | All content CRUD, manage users, configure webhooks and API tokens |
| **Editor** | Content creator | Create, edit, and publish content entries; manage own content |
| **Viewer** | Read-only access | View content and media, no editing capabilities |

> ⚠️ **Note:** Role assignments are managed by administrators. Contact your Super Admin or Admin if you need elevated permissions.
